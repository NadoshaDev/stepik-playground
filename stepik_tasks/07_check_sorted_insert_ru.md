# Задача 07 — Проверка сортировки после вставки

## Цель

Вставить число в отсортированный список и проверить, сохранилась ли сортировка.

## Что я отрабатывала

* метод insert()
* range(len(list) - 1)
* сравнение соседних элементов
* логические переменные
* оператор break

## О чём я думала во время решения

Сначала мне хотелось проверять сортировку прямо внутри цикла и сразу выводить результат.

Но довольно быстро стало понятно, что одна удачная проверка ещё ничего не говорит о всём списке. Нужно проверить все соседние элементы и только потом делать вывод.

Отдельно пришлось разобраться с идеей логического флага. До этой задачи мне было не очень понятно, зачем нужна переменная вроде `is_sorted`, если можно просто писать условия.

Во время обсуждения стало понятно, что такая переменная позволяет программе "помнить", встретилось ли нарушение сортировки раньше.

Также интересным оказался вопрос про `break`. Я поняла, что после первого нарушения порядка продолжать проверку уже нет смысла.

## Вывод

Самым полезным в этой задаче оказалось не использование метода `insert()`, а понимание того, как проверять состояние списка целиком.

Эта задача помогла лучше понять:

* сравнение соседних элементов;
* работу логических флагов;
* использование `break`;
* идею остановки алгоритма, когда ответ уже найден.

Для меня это была одна из первых задач, которая ощущалась как небольшой алгоритм, а не просто работа со списком.

## 🐍 Код
```python
nums = [4, 15, 19, 35, 53, 64]

number = int(input())
index = int(input())

nums.insert(index, number)

is_sorted = True

for i in range(len(nums) - 1):
    if nums[i] > nums[i + 1]:
        is_sorted = False
        break

if is_sorted:
    print("Сортировка сохранилась")
else:
    print("Список уже не отсортирован")
