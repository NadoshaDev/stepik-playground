#  Мини-проект: Система записи клиента

> Учебный проект: реализация логики выбора свободного времени (аналог записи к мастеру).  
> Разрабатывался 2 дня. Главная цель — не просто написать код, а понять процесс, ошибки и защиту от сбоев.

---

##  Общая логика проекта

Есть:
- Список всех доступных слотов (`all_times`)
- Список занятых слотов (`busy_times`)

Нужно:
1. Получить свободные слоты
2. Показать пользователю
3. Дать выбрать время
4. Проверить ввод
5. Записать пользователя

---

##  День 1 — Базовая логика

###  Версия 1: Получение свободных слотов
 Первая задача — просто понять работу списков и фильтрации.

```python
all_times = ["10:00", "11:00", "12:00", "13:00"]
busy_times = ["11:00"]
free_times = []

for time in all_times:
    if time not in busy_times:
        free_times.append(time)

print("Свободные слоты:")
for time in free_times:
    print(time)
```

 **Что я поняла:**
- Как работают списки и `for`
- Как фильтровать данные через `if not in`

---

###  Версия 2: Ввод времени вручную
 Пользователь вводит время сам (например `"10:00"`).

```python
chosen_time = input("Выберите время: ")

while chosen_time not in free_times:
    print("Ошибка, такого времени нет")
    chosen_time = input("Выберите время: ")

print("Вы записаны на:", chosen_time)
```

️ **Проблемы:**
- Неудобно вводить `"10:00"` вручную
- Легко ошибиться в формате
- Не подходит для реального продукта

 **Вывод:** нужно упростить ввод для пользователя.

---

##  День 2 — Интерфейс и валидация

###  Версия 3: Переход на выбор через номера
 Заменила ввод времени на выбор через цифры.

```python
for i in range(len(free_times)):
    number = i + 1
    print(f"{number} - {free_times[i]}")

chosen_number = input("Выберите номер: ")
```

 **Логика:** `номер → индекс → время`

---

###  Версия 4: Первая попытка проверки (ошибочная)
 Пыталась сделать проверку сразу в одном условии.

```python
while not chosen_number.isdigit() and chosen_number not in free_times:
    print("ошибка")
```

 **Проблемы:**
- Логика не работала
- Путаница между `and` и `or`
- Сравнение разных типов (число vs время)

 **Вывод:** нельзя проверять всё сразу.

---

###  Версия 5: Попытка объединить всё в один while
 Хотела сделать «красиво» в одну строку.

```python
while not chosen_number.isdigit() or not 0 < int(chosen_number) <= len(free_times):
    print("ошибка")
```

 **Проблемы:**
- Сложно понять и отладить
- Путаница в логике условий

 **Вывод:** упрощение синтаксиса может усложнить понимание.

---

###  Версия 6: Финальное решение (разделённые проверки)
 Разбила валидацию на два шага.

```python
chosen_number = input("Нажмите цифру, соответствующую времени: ").strip()

# 1. Проверка: это число?
while not chosen_number.isdigit():
    print("Ошибка: введите только цифру.")
    chosen_number = input("Нажмите цифру, соответствующую времени: ").strip()

# 2. Проверка: входит в диапазон?
while not 0 < int(chosen_number) <= len(free_times):
    print("Ошибка: цифры нет в списке.")
    chosen_number = input("Нажмите цифру, соответствующую времени: ").strip()

# 3. Запись
i = int(chosen_number) - 1
chosen_time = free_times[i]
busy_times.append(chosen_time)

print(f"Вы записаны на: {chosen_time}")
```

 **Ключевые выводы:**
- Сначала проверяется тип данных (`isdigit`), потом диапазон
- Нельзя делать `int()` до проверки на цифры
- Лучше разделять логику, чем лепить всё в одну строку
- `.strip()` спасает от случайных пробелов
